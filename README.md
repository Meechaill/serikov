# Serikov Coffee — сайт

Собранная (production) версия лендинга Serikov Coffee.
Исходники (React/Vite-проект) в этой папке отсутствуют — здесь только готовая сборка:
`assets/app.js` — минифицированное React-приложение, `assets/app.css` — стили.

Макет: [Figma](https://www.figma.com/design/2YJUMNac0vA4ceytlBF82S/Untitled?node-id=8-2)

## Как запустить

Открывать `index.html` двойным кликом нельзя — пути вида `/assets/...`
требуют локальный сервер. Любой из способов:

**Node.js (рекомендуется):**
```
npm install
npm start
```
Сайт: http://localhost:3000

**Альтернативы:** `npx serve .` или `python -m http.server 8000` из корня папки.

## Структура

```
index.html                  — точка входа
server.js                   — простой express-сервер для локального запуска
assets/
  app.js                    — JS-бандл приложения (минифицирован)
  app.css                   — CSS-бандл
  hits.js                   — секция «Хиты продаж» (читаемый исходник; врезает
                              секцию между каталогом и «путём кофе» и сдвигает
                              всё, что ниже)
  images/
    logo.svg                — логотип в шапке
    hero/                   — первый экран: зёрна, упаковки кофе, декоративные подложки
                              (decor-left/right.svg — составные: внутри 20
                              позиционированных PNG-частей каждая)
    hits/                   — упаковки товаров секции «Хиты продаж»
                              (originals/ — исходники без цветных клякс)
    catalog/                — секция каталога «Для дома / Для бизнеса / Для бариста»:
                              product-*  — фото продуктов
                              card-*     — фоны/фигуры карточек
    journey/                — секция «путь кофе» (шаги 01–03 и CTA):
                              plantation-* — «Плантации»
                              selection-*  — «Ручной отбор»
                              flavor-*     — «Раскрытие вкуса»
                              quality-*    — «Контроль качества»
                              cta-*        — блок «Выбрать идеальный кофе»
                              road-*       — боковые «дороги» вдоль секции
```

## Адаптация под ПК

Макет — фиксированный холст 1440px (`#main-container`), который масштабируется
под ширину окна:

Всю адаптацию делает скрипт в `index.html` (экраны от 768px):

- CSS `zoom` подбирается так, чтобы первый экран (до низа бегущей строки,
  ~1040px макета) влезал по высоте, но не шире, чем позволяет окно.
- Полей по бокам не остаётся: холст (`#main-container`) расширяется до ширины
  окна, центрированный контент сдвигается в середину (`margin-left`,
  не `transform` — transform на предке ломает `position:fixed` у модалки),
  а декор, прижатый в макете к краям холста, остаётся у краёв окна.
  Полноэкранные подложки (бегущая строка, эллипс, фоны секций) растягиваются;
  контенту бегущей строки при нехватке ширины дублируются элементы,
  чтобы цикл анимации оставался бесшовным.
- Собственное масштабирование `app.js` (для окон уже 1440px) нейтрализовано:
  бандл пропатчен читать ширину макета из `window.__layoutWidth`, который
  задаёт скрипт. Ниже 768px скрипт не вмешивается и работает старая логика
  бандла. Также в бандле исправлен несуществующий Tailwind-класс
  `md:origin-top-center` (CSS-правило в `index.html`).

Мобильная адаптация (<768px) пока не делалась.

## Важно при правках

- Пути к картинкам захардкожены внутри `assets/app.js`. При переименовании
  файла в `assets/images/` нужно заменить строку пути и в `app.js`.
- Кнопки сайта нарисованы кодом: div-подложка со скосом 12px при высоте 54
  и скруглением 20 (`border-radius: 20px; transform: skewX(-12.53deg)`),
  текст поверх. SVG-файлы кнопок удалены.
- Для серьёзных правок дизайна/текста нужны исходники проекта (src),
  их в этой папке нет.
